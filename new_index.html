Overview & Philosophy

This portfolio captures my academic journey through the MS in Business Analytics program at Baruch College’s Zicklin School of Business. It integrates lessons from three core courses—Database Management Systems (CIS 9340), Programming for Analytics (CIS 9650), and Managerial Statistics (STA 9708)—and showcases how these subjects interconnect to form a unified analytical mindset.

My goal is to demonstrate not just mastery of individual topics, but an appreciation for the ways that database design, programming skills, and statistical reasoning reinforce each other. By presenting conceptual maps, case studies, cheat sheets, timelines, and reflections, I aim to create a professional and intellectually rich artifact that can be shared with peers, recruiters, and professors.

Projects & Case Studies

Below is a simplified demonstration of a project typical of our coursework. Using Python and a sample dataset, I built a linear regression model to explore the relationship between two variables. This process illustrates how database extracts (e.g., from MySQL) can be loaded into Pandas, analyzed, and visualized.

# sample Python code
import pandas as pd
import numpy as np
from sklearn.linear_model import LinearRegression

# assume data is loaded from a SQL query into a DataFrame
# df = pd.read_sql("SELECT x, y FROM my_table", connection)

# for demonstration, create synthetic data
df = pd.DataFrame({{
    'x': np.arange(0, 10.5, 0.5),
    'y': 2 * np.arange(0, 10.5, 0.5) + np.random.normal(0, 2, len(np.arange(0, 10.5, 0.5)))
}})

model = LinearRegression()
model.fit(df[['x']], df['y'])
print('Slope:', model.coef_[0])
print('Intercept:', model.intercept_)

Results: slope ≈ 1.60, intercept ≈ 1.55. These values show the best‑fit line describing the relationship between x and y.

Statistics Cheat Sheet

A quick reference for core statistical concepts covered in STA 9708. Use this section as a refresher when working on analyses and projects.

  • Probability rules: P(A ∪ B) = P(A) + P(B) − P(A ∩ B); P(A | B) = P(A ∩ B) / P(B)
  • Binomial distribution: Mean = n·p, Variance = n·p·(1 − p)
  • Normal distribution: The 68‑95‑99.7 rule (empirical rule) describes data within 1, 2, and 3 standard deviations of the mean
  • Confidence interval for the mean: x̄ ± zα/2 · (σ / √n) for known σ, or t‑based interval when σ is unknown
  • Hypothesis testing: Test statistic z = (x̄ − μ0) / (σ / √n). Reject H0 if |z| > zα/2
  • Simple regression: y = β₀ + β₁x + ε; use t‑tests to evaluate β₁ and R² to measure goodness‑of‑fit

Analytics Ecosystem & Reflection

Together, these courses build a robust analytical toolkit. Learning to structure and query databases ensures data integrity; programming empowers automation and exploration; and statistics provide methods to quantify uncertainty and test hypotheses. The projects, notes, and timelines presented here demonstrate how these domains interact in practice.

I view my MSBA journey as a system where inputs (datasets, assignments, lectures) flow through processes (analysis, modelling, collaboration) to produce outputs (insights, reports, applications). Inspired by systems thinking resources, I strive to iterate on this system: enhancing data pipelines, refining code quality, and deepening statistical reasoning.

Looking ahead, I plan to expand this portfolio with more case studies (e.g., time series forecasting, database normalization with triggers and views), add interactive widgets for practice, and share blog posts reflecting on how these experiences shape my path toward data-driven leadership.
